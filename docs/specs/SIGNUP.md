## 1. 입력 항목

| 필드 | 설명 |
|------|------|
| 이메일 | 로그인 ID |
| 닉네임 | 2~20자 (`maxLength={20}`) |
| 비밀번호 | 8자 이상, 특수문자 1개 이상 |
| 비밀번호 확인 | 비밀번호 재입력 |

---

## 2. 검증

### 2-1. 방식

- **브라우저 기본 검증 사용하지 않음** (`noValidate`, `required`/`minLength` 없음)
- **폼 하단 인라인 에러 없음**
- 모든 피드백은 **토스트**로만 표시
- 검증은 JavaScript로 순차 수행. **현재 단계 실패 시 다음 단계로 진행하지 않음**

### 2-2. 검증 순서

| 순서 | 항목 | 실패 사유 예시 |
|------|------|----------------|
| 1 | 이메일 입력·형식 | `이메일을 입력해주세요.` / `올바른 이메일 형식이 아닙니다.` |
| 2 | 닉네임 | `닉네임은 2~20자로 입력해주세요.` |
| 3 | 비밀번호 강도 | `8자 이상, 특수문자 1개 이상 포함해야 합니다.` |
| 4 | 비밀번호 일치 | `비밀번호가 일치하지 않습니다.` |
| 5 | ID(이메일) 중복 | `이미 사용 중인 이메일입니다.` |

### 2-3. 이메일 중복 감지

**클라이언트에서 `auth.users` 직접 조회는 불가** (Supabase Auth 스키마는 서버 전용).

`signUp` API 응답으로 판별한다 (Supabase 권장):

| 응답 | 처리 |
|------|------|
| `error` | API 오류 매핑 |
| `user` 없음 | 회원가입 실패 |
| `user.identities.length === 0` | **이미 등록된 이메일** (열거 방지를 위해 HTTP error 대신 이 형태로 올 수 있음) |
| `user.identities` 1개 이상 | 신규 가입 성공 |

### 2-4. 비밀번호 강도

- 8자 이상
- 특수 문자 1개 이상 (`!@#$%^&*()` 등)

## 3. 성공 / 실패

### 3-1. 실패

토스트 (error):

| title | description |
|-------|-------------|
| `회원가입 실패` | `{실패 사유}` |

API 오류 등 매핑되지 않은 경우 description:

`회원가입에 실패했습니다. 잠시 후 다시 시도해주세요.`

### 3-2. 성공

1. Supabase `signUp` 호출 → `profiles` 행 자동 생성 (DB 트리거)
2. 토스트 (success):

| title | description |
|-------|-------------|
| `회원가입 성공` | `가입하신 이메일로 인증 링크를 보냈습니다. 인증 후 로그인해주세요.` |

3. **로그인 화면**으로 전환
   - 비밀번호, 비밀번호 확인, 닉네임 필드 초기화
   - 이메일 input 값은 유지

---

## 4. 이메일 인증 (Supabase Cloud)

- **Confirm email ON** (클라우드 기본) 전제
- 가입 직후 **세션 없음** → 자동 로그인 없음
- 인증 링크 클릭 후 로그인

---

## 5. UI / UX (명세 외 구현)

| 항목 | 동작 |
|------|------|
| 제출 중 | 버튼 `처리 중...` + `disabled` |
| 로그인 ↔ 회원가입 전환 | 하단 토글, 비밀번호·닉네임 초기화 |
| 닉네임 | HTML `maxLength={20}` (입력 길이 제한) |
