# 확인 Dialog 명세

> 인덱스: [README.md](./README.md) · 삭제: [EVENT_DELETE.md](./EVENT_DELETE.md) · 수정: [EVENT_UPDATE.md](./EVENT_UPDATE.md)

삭제·반복 범위 선택 등 **되돌리기 어렵거나 분기가 필요한 동작**의 확인 UI.  
**브라우저 `window.confirm()`은 사용하지 않는다.**

---

## 1. 역할

- 앱 Modal과 **동일한 톤**의 커스텀 오버레이
- 제목 + 설명(선택) + **버튼 2~3개**
- 오버레이 클릭·`취소` 는 **취소(닫기)** 로 동작
- **액션 버튼**(`삭제`, `해당 일정만`, `전체 일정` 등)은 `onClick`만 실행하고 **`onClose`는 호출하지 않는다** — Dialog 닫기·후속 처리는 각 `onClick`/부모 상태로 처리
- **달력과 AI 어시스턴트가 동일한 버튼 구성**을 사용한다

---

## 2. 삭제 확인

범위 선택과 확인을 **하나의 Dialog**로 통합한다 (별도 범위 Dialog → Confirm 2단계 아님).

| 케이스 | 버튼 | 안내 문구 (예) |
|--------|------|----------------|
| **개별 일정** (비반복·분리된 단일) | `취소` / `삭제` | `이 일정을 삭제하시겠습니까?` |
| **반복 일정** (2회차 이상 남음) | `취소` / `해당 일정만` / `전체 일정` | `반복 일정을 어떻게 삭제할까요?` |
| **반복 마지막 1회차** (유한 반복) | `취소` / `전체 삭제` | `전체 반복 일정을 삭제하시겠습니까?` |

- `삭제` / `전체 삭제` / `전체 일정` 은 파괴적 액션 스타일(빨강).
- 무한 반복(계속 반복)은 마지막 1회차 케이스 **아님** → 2회차 이상과 동일.

---

## 3. 수정 범위 선택

반복 일정 **수정**은 확인이 아니라 **범위 선택**이지만 동일한 Dialog 톤을 쓴다.

| 케이스 | 버튼 |
|--------|------|
| **개별 일정** | Dialog 없음 (바로 저장) |
| **반복 일정** | `취소` / `해당 일정만` / `전체 일정` |

- 파괴적이지 않으므로 일반 스타일.
- 「해당 일정만」 = 제외 + 개별 분리, 「전체 일정」 = 시리즈 수정 ([EVENT_UPDATE.md](./EVENT_UPDATE.md)).

---

## 4. 모바일

- 버튼 3개는 **세로 스택** 가능. 라벨은 짧게(`해당 일정만`, `전체 일정`).

---

## 5. 구현 상태

| 항목 | 상태 |
|------|------|
| 커스텀 Confirm Dialog (`actions[]` 다중 버튼, danger 스타일) | ✅ 구현 |
| 개별 삭제 2버튼 (취소/삭제) | ✅ 구현 (달력·AI) |
| 반복 삭제 통합 3버튼 (취소/해당/전체, 범위+확인 일원화) — **달력** | ✅ 구현 (2단계 → 단일 Dialog) |
| 반복 마지막 1회차 2버튼 (취소/전체 삭제) — **달력** | ✅ 구현 |
| AI 반복 삭제 「해당 일정만」 (3버튼 완성) | ✅ 구현 |
| 반복 수정 범위 3버튼(취소/해당/전체) — 달력+AI | ✅ 구현 |
